# Internationalization scaffold (i18n / l10n)

Status: **C01 L16** — Phase-0 **English-only** message catalog plus a lookup
helper. This is a scaffold, not full localization.

SessionLedger UI copy, CLI help, and operator docs remain English. Multi-locale
catalogs, fluent/ICU plural rules, and runtime locale switching are deferred
until product demand justifies the dependency cost.

Related: [`docs/DESIGN.md`](../DESIGN.md) (product scope), ADR trail under
[`docs/adr/`](../adr/) for larger product decisions.

## Phase-0 decision

| Choice | Rationale |
|--------|-----------|
| Tiny JSON catalog (`locales/en.json`) | Cheap, readable, no new crates beyond existing `serde_json` |
| Rust helper (`src/i18n.rs`) | `t(key)` + `try_catalog(locale)` with English fallback hooks |
| Defer fluent / gettext / ICU | Full i18n runtimes are effort **L**; score-0 gap closes with a real catalog stub first |
| English-only shipping locale | Matches current CLI/viewer/docs; additional `locales/*.json` files are the extension point |

This is **not** a claim that the viewer or CLI already route every user-visible
string through the catalog. Inline English strings remain common; the catalog
establishes the SSOT shape and lookup API for incremental adoption.

## Catalog layout

```
locales/
  en.json          # Phase-0 default (embedded via include_str!)
  # fr.json        # future: add file + register in try_catalog / loader
```

Each catalog file:

```json
{
  "locale": "en",
  "messages": {
    "viewer.app_title": "SessionLedger",
    "cli.app_about": "..."
  }
}
```

Keys use dotted namespaces (`cli.*`, `viewer.*`). Lookup:

- `session_ledger::i18n::t("viewer.app_title")` → message text
- Missing key → returns the key itself (stable for incomplete migrations)
- `try_catalog("fr")` → `None` today (future hook)

## Future hooks

When multi-locale becomes a goal:

1. Add `locales/<tag>.json` with the same key set (or a documented subset).
2. Extend `try_catalog` / add a small loader that maps BCP-47 tags → catalogs.
3. Optionally adopt Fluent (`.ftl`) or ICU MessageFormat for plurals/selects.
4. Wire CLI (`clap`) and viewer surfaces to `t(...)` incrementally; keep
   English as the fallback chain: `requested → en → key`.
5. Consider `SL_LOCALE` / OS locale detection only after catalogs exist.

Do **not** add gettext extract pipelines or Crowdin/Weblate wiring until at
least one non-English catalog ships.

## Machine verification (SelfCheck)

Hermetic catalog + helper smoke (no daemon, no network, no fluent runtime):

```powershell
pwsh ./scripts/i18n-check.ps1 -SelfCheck
```

`-SelfCheck` asserts `locales/en.json` exists with required keys, `src/i18n.rs`
exposes `t` / `try_catalog` / `DEFAULT_LOCALE`, this doc keeps the Phase-0
decision and future-hooks section, and the Rust wrapper test is present.
